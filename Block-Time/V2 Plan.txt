  Phase Details 
                                         
  Phase 1: Foundation
  Goal: SwiftData schema, BlockTimeKit package, and migration service proven safe before any UI or TestFlight build touches them.
  Success criteria:                                                                                                                                  
  1. Real v1 production .sqlite migrates without data loss — time fields round-trip String → TimeInterval correctly
  2. Simulated mid-migration crash retries on relaunch without duplicating or corrupting records                                                     
  3. SwiftUI preview opens on InMemoryFlightRepository with no CloudKit connection required     
  4. App Group store URL pinned — widget add/remove does not produce an empty store                                                                  
                                                                                                                                                     
  Phase 2: Calculators & Tests                                                                                                                       
  Goal: Every business rule and parser is pure function with 100% test coverage before any UI wires against it.                                      
  Success criteria:                                                                                                                                  
  1. Full FRMS, night time, and time converter suite passes without a simulator
  2. All four FRMS rule sets have at least one test that catches a known edge case before the fix                                                    
  3. CSV, ACARS, and roster fixture tests pass including known bad inputs                        
  4. InMemoryFlightRepository tests cover all protocol operations without SwiftData                                                                  
                                                                                                                                                     
  Phase 3: Core UI                                                                                                                                   
  Goal: Every v1 screen exists at full feature parity, @Observable ViewModels, backed by repository protocol.                                        
  Success criteria:                                                                                                                                  
  1. Add/edit/delete flight workflow works end-to-end on device against migrated store
  2. FRMS view outputs match v1 for identical input data                                                                                             
  3. No screen contains an ObservableObject ViewModel   
  4. All major screens have working SwiftUI previews without CloudKit                                                                                
                                                                                                                                                     
  Phase 4: Import Pipeline                                                                                                                           
  Goal: CSV, ACARS, roster imports complete end-to-end; merge review sheet correctly surfaces duplicates.                                            
  Success criteria:                                                                                                                                  
  1. CSV import shows field-level diffs in merge review sheet; committing persists all records                                                       
  2. ACARS photo populates Add Flight form correctly for all four aircraft types                                                                     
  3. Roster import creates planned flights visible immediately — no restart required
  4. Flights list refreshes after background import without manual pull-to-refresh                                                                   
                                                                                                                                                     
  Phase 5: Widgets & Extensions                                                                                                                      
  Goal: WidgetKit next-flight widget and App Intents read from SwiftData App Group container.                                                        
  Success criteria:                                                                                                                                  
  1. Widget shows correct next flight from SwiftData App Group store
  2. Widget shows correct empty state — no crash, no stale data                                                                                      
  3. Midnight-crossing flights attributed to the correct date  
                                                                                                                                                     
  Phase 6: Export & Settings                                                                                                                         
  Goal: PDF/CSV/.ics exports work against SwiftData repository; all settings have one consolidated source of truth.                                  
  Success criteria:                                                                                                                                  
  1. PDF export is visually identical to v1 with correct columns and totals
  2. CSV export re-imports cleanly with no field misalignment                                                                                        
  3. Setting change on one device appears on second device within 60 seconds
  4. No service or ViewModel reads @AppStorage directly                                                                                              
                                                                                                                                                     
  Phase 7: Mac + Pre-release
  Goal: Mac target builds from BlockTimeKit; full migration rehearsal passes on a real v1 device.                                                    
  Success criteria:                                                                                                                                  
  1. Mac target compiles and runs Add/Edit/Delete and Settings from BlockTimeKit                                                                     
  2. Real device with full v1 logbook updates to v2.0 — zero records lost                                                                            
  3. CloudKit syncs migrated data to a second device within 60 seconds
