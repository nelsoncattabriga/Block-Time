```swift
    func fetchAll() -> [CrewContactEntity] { ... }

    /// Finds contact by name (case-insensitive). Returns nil if not found.
    func fetchContact(name: String) -> CrewContactEntity? { ... }

    /// Creates or updates a contact. If a contact with this name exists, updates notes.
    /// name must be non-empty (guard at app layer — Core Data attribute is optional="YES").
    func upsert(name: String, notes: String) { ... }

    /// Deletes the contact with the given name. No-op if not found.
    func delete(name: String) { ... }

    /// Returns all contacts as CrewContactBackup array for CSV export.
    /// Called on main thread before background dispatch in AutomaticBackupService.
    func fetchAllAsBackup() -> [CrewContactBackup] { ... }
}
```

Implementation rules:
- All methods use `FlightDatabaseService.shared.viewContext.performAndWait { }` for all Core Data access.
- `fetchAll()` and `fetchContact(name:)` use `NSFetchRequest<CrewContactEntity>` with appropriate sort/predicate.
- `fetchContact` predicate: `NSPredicate(format: "name ==[c] %@", name)` (case-insensitive).
- `upsert`: if fetchContact returns non-nil, update notes + modifiedAt; else create new entity with `UUID()` id and current Date() for both timestamps.
- After any write, call `try? FlightDatabaseService.shared.viewContext.save()` inside the `performAndWait` block.
- Use `guard !name.trimmingCharacters(in: .whitespacesAndNewlines).isEmpty else { return }` in `upsert`.
- Swift 6: class is `@MainActor` — no `nonisolated` needed; `performAndWait` is synchronous on main thread.
