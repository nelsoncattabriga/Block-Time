Two edits to `Block-Time/ViewModels/BulkEditViewModel.swift`:

**Edit 1 — `setupModificationTracking()` (around lines 454-458):**

Replace:
```swift
$customCounterStates
    .sink { [weak self] _ in
        self?.checkForModifications()
    }
    .store(in: &cancellables)
```

With:
```swift
$customCounterStates
    .receive(on: RunLoop.main)
    .sink { [weak self] newStates in
        self?.checkForModifications(customStates: newStates)
    }
    .store(in: &cancellables)
```

Reason: `@Published` fires via `willSet`, so reading `self.customCounterStates` inside the sink returns the OLD value. Using the closure parameter `newStates` gets the value the user just typed. `.receive(on: RunLoop.main)` is added for consistency with the `$blockTimeRole` sink pattern already in this file (around line 395).
