# Quick Task 260601-a73: Add Crew Notes Management UI to Settings Summary

**One-liner:** Crew Notes card in Crew & Ops Data settings with list/edit push navigation backed by CrewContactService.

## What Was Built

**CrewNotesManageView.swift** (new file):
- `CrewNotesManageView` — `NavigationStack` with a `List` of `CrewContactEntity` rows loaded from `CrewContactService.shared.fetchAll()`. Each row shows name in `.subheadline` weight and a 60-char notes preview in `.caption .secondary`. Empty state shows "No crew notes saved." centered. Navigation destination pushes `CrewNoteEditView`.
- `CrewNoteEditView` — `Form` with read-only Name section and editable Notes `TextEditor` (minHeight 120). Cancel/Save toolbar items on `.topBarLeading`/`.topBarTrailing`. Destructive "Delete Note" button below the form triggers a `confirmationDialog` before calling `CrewContactService.shared.delete(name:)`. Save calls `upsert(name:notes:)` then dismisses.

**SettingsView.swift** (modified):
- Added `ModernCrewNotesCard` private struct after `ModernCustomFieldsCard`. Card matches existing card styling (thinMaterial, corner radius 12, blue overlay border). Manage button sets `showManage = true`; `.navigationDestination(isPresented: $showManage)` pushes `CrewNotesManageView`.
- Added `ModernCrewNotesCard()` to `PersonalCrewSettingsView.body` after `ModernCustomFieldsCard()`, before `Spacer(minLength: 20)`. No existing cards removed or reordered.
