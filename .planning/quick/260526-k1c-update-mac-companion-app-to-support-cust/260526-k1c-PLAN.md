<objective>
Bring the Block-Time Mac companion app to parity with iOS's custom-fields system. iOS replaced the legacy single `customCount: Int` counter with ten per-definition slots (`counter1`–`counter10`, all `String?` in Core Data) driven by `CustomCounterDefinition` objects synced via iCloud KVS. The Mac app has no knowledge of this, so iOS-entered custom data is invisible on Mac, the edit panel shows the wrong control, and the table shows the retired int.

Purpose: a pilot who logs custom fields on iPhone must see and edit those fields on Mac.

Output: two new Mac-only files (model copy + KVS-aware service) plus edits to the data layer, table columns, edit panel, and sync indicator.
</objective>

The Mac target CANNOT import the iOS target. Copy these types verbatim (identical Codable keys
are mandatory so JSON round-trips with iOS).

From Block-Time/Models/CustomCounterDefinition.swift (source of truth — copy faithfully):
```swift
enum CounterType: String, Codable, CaseIterable, Identifiable {
    case time, decimal, integer, text
    var id: String { rawValue }
    // displayName / subtitle optional on Mac — only add if used
}
struct CustomCounterDefinition: Codable, Identifiable, Hashable {
    let columnIndex: Int   // 1–10
    var label: String
    var type: CounterType
    var showTotal: Bool
    var id: Int { columnIndex }
    // custom init(from:) uses decodeIfPresent for showTotal (default true) — REQUIRED for forward-compat
    enum CodingKeys: String, CodingKey { case columnIndex, label, type, showTotal }
}
```

iCloud KVS WIRE FORMAT (verified in Block-Time/Services/CloudKitSettingsSyncService.swift):
- Key: `"cloud_customCounterDefinitions"`
- Value is a JSON **String** (not Data): iOS does
  `String(data: JSONEncoder().encode([CustomCounterDefinition]), encoding: .utf8)`
  then `ubiquitousStore.set(definitionsJSON, forKey: ...)`.
- To READ on Mac: `NSUbiquitousKeyValueStore.default.string(forKey:)` →
  `.data(using: .utf8)` → `JSONDecoder().decode([CustomCounterDefinition].self, from:)`.
- Local UserDefaults key `"customCounterDefinitions"` stores JSON **Data** (JSONEncoder output, no String wrap).

Existing column system (do NOT break — verified callers):
- `LogbookColumn.scrollingColumns(hhmm:rounding:localTime:useIATA:)` is the SINGLE source of truth for
  the canonical column set. It is called from FIVE sites:
  ColumnPreferences.init (builds `order`), ColumnPreferences.visibleColumns (builds lookup),
  ColumnPreferences.reset, ColumnManagerPopover.columnList, and MacLogbookTableView itself.
  Therefore custom columns MUST be generated INSIDE scrollingColumns so order/visibility/persistence
  all work with zero changes to ColumnPreferences and ColumnManagerPopover.
- `MacFlightRow.parseTime`, `MacFlightRow.formatTime`, `MacFlightRow.decimalDisplay` are reused for typed display.

Mac entitlements already include `com.apple.developer.ubiquity-kvstore-identifier` and iCloud container
`iCloud.com.thezoolab.blocktime`. No entitlement changes needed.
