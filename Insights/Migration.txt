  attributes including booleans and datesâ€”JSON prevents lossy conversion.

  JSON Structure Design: The migration package uses a nested structure with clear separation of concerns: MigrationPackage contains metadata 
  (counts, versions), flights array, aircraft array, settings object, and preferences object. This design makes it easy for Block-Time's     
  importer to selectively process each section and provide granular progress feedback.                                                       
                                                                                                                                             
  File Extension Choice: Using .blocktime instead of .json serves multiple purposes: (1) prevents users from accidentally opening it in text 
  editors, (2) allows Block-Time to register as the handler for this file type, (3) makes the file's purpose immediately clear, and (4)      
  enables "Open In..." functionality where users can select Block-Time directly from Files app.                                              
                                                                                                                                             
  Date Encoding Strategy: ISO 8601 format ensures dates are preserved with timezone information across the migration. Core Data stores dates 
  as Date objects, but JSON needs a string representationâ€”ISO 8601 is unambiguous and supported by all modern JSON parsers, preventing the   
  common timezone conversion bugs that plague migrations.
